üìã System Requirements

Windows 10/11, macOS 10.15+, or Ubuntu 18.04+
Webcam (720p or higher recommended)
Python 3.8 or higher
4GB RAM minimum (8GB recommended)
Dedicated GPU (optional but recommended for better performance)

üöÄ Installation Steps

1

Install Python Dependencies

Open a terminal and run the following command:

pip install -r requirements.txt
2

Launch the Application

Start the web interface with:

python run_app.py

This will start the server at http://localhost:5000

3

Open in Browser

Navigate to http://localhost:5000 in your web browser to access the control interface.

üìù Note: The system will automatically create a basic model if one doesn't exist. For best results, use the training interface to train the model with your own gestures.

üéì Training Your Own Gestures

To improve recognition accuracy or add custom gestures, follow this training process:

1

Launch Training Interface

Run the application and navigate to the training page:

python run_app.py

Then open http://localhost:5000/training in your browser

2

Select Gesture to Train

The interface will show a list of gestures. Select one to train.

3

Perform the Gesture

When prompted, perform the gesture clearly in view of the camera.

üí° Tip: Perform the gesture from multiple angles and slightly different variations to improve recognition.

4

Repeat for Multiple Samples

Provide at least 20-30 samples of each gesture for better accuracy.

5

Train the Model

After collecting samples, the system will automatically train the model.

üìù Note: Training may take several minutes depending on your system performance.

6

Test the Recognition

Use the test mode to verify the system correctly recognizes your gestures.

üìä Training Best Practices

1

Lighting Conditions

Train in the same lighting conditions where you'll be using the system.

2

Background

Use a consistent, non-distracting background during training.

3

Distance from Camera

Maintain a consistent distance from the camera during training and use.

4

Gesture Variation

Include slight variations of each gesture to make the system more robust.

5

Regular Retraining

Retrain the system periodically to maintain accuracy, especially if lighting conditions change.

üéØ Creating Custom Gestures

To add completely new gestures to the system:

1

Edit Gesture List

Add your new gesture name to the gestures array in accessibility_controller.py

2

Define Action

Add the corresponding action in the execute_gesture_action method.

3

Train the Gesture

Use the training interface to collect samples for your new gesture.

4

Test Thoroughly

Test the new gesture extensively to ensure it doesn't conflict with existing gestures.

üîß Common Issues and Solutions

1

Camera Not Detected

Solution: Check if another application is using the camera. Restart the application.

2

Poor Gesture Recognition

Solution: Improve lighting conditions, retrain gestures, or adjust sensitivity settings.

3

System Running Slowly

Solution: Close other applications, reduce camera resolution, or use a more powerful computer.

4

Commands Not Executing

Solution: Check if the system is in the correct mode (cursor vs gesture mode).

5

Web Interface Not Loading

Solution: Ensure the server is running on port 5000 and no other application is using that port.

üìù Performance Optimization

1

Adjust Camera Resolution

Lower resolutions will improve performance on slower systems.

2

Reduce Processing Rate

Decrease the processing frames per second in settings for better performance.

3

Simplify Gestures

Use simpler, more distinct gestures for faster recognition.

4

Use GPU Acceleration

Enable GPU support in the settings if you have a compatible graphics card.
